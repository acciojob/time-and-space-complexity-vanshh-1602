Time, Space, Complexity

Time Complexity measures how the execution time of an algorithm increases with input size and is crucial for evaluating how efficient the algorithm is. Efficient algorithms aim to minimize time complexity to ensure faster performance for larger datasets. Common time complexities include O(1) (constant time), which is highly efficient, O(log n) (logarithmic time), which scales well, and O(n) (linear time), where execution time increases proportionally with input size. Less efficient complexities, such as O(n²) (quadratic time) or O(2ⁿ) (exponential time), can cause significant delays for large inputs.

Space Complexity measures the memory usage of an algorithm relative to the input size. O(n)
